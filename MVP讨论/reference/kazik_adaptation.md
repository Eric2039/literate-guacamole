# kazik_adaptation.md — 数字生命卡兹克 skill 的选择性借用

> 卡兹克 (Khazix) 的写作 skill 完整内容在 `KazikSkillReference/SKILL.md`。**那是参考资料, 不是直接套用**。卡兹克写公众号长文 (4000-8000 字, 不加小标题, 个人品牌姿态), ZIXEL SEO 写关键词驱动文章 (1500+ 字, 必须 H2/H3, 子虔小编姿态)。形式和目标都不同。
>
> 这个文件只列三件事: 哪些元素**可以迁移**, 哪些**不能迁移 (说明理由)**, 以及可迁移元素在 SEO 项目里**整合到哪些文件**。

---

## 1. 可迁移元素 (拿来用, 适配后)

### 1.1 L1 禁用词清单 (绝对禁用)

这些词 AI 痕迹极重, 一出现立刻暴露。SEO 文章和公众号文章都禁用:

| 禁用词 | 替换方向 |
|---|---|
| 说白了 | 坦率的讲 / 其实就是 |
| 意味着 / 这意味着 | 那结果会怎样呢 / 所以呢 |
| 本质上 | 说到底 / 其实 |
| 换句话说 | 你想想看 / 也就是说 |
| 不可否认 | 直接删, 改正面陈述 |
| 综上所述 / 总的来说 | 具体的回扣句 |
| 首先...其次...最后 | 自然转场词 |
| 值得注意的是 / 不难发现 | 直接说 |
| 让我们来看看 / 接下来让我们 | 直接进入内容 |
| 在当今...的时代 | 删, 从具体事件切入 |
| 随着...的发展 | 删, 从具体事件切入 |

### 1.2 L1 禁用标点

| 标点 | 替换 |
|---|---|
| 冒号 `:` (中文全角 / 半角) | 用逗号代替 |
| 破折号 `——` | 用逗号或句号代替 |
| 双引号 `""` 或 `""` | 用 `「」` 代替, 或直接不加引号 |

注: SEO 文章的 H2/H3 标题本身可以保留正常标点结构 (例如 `## XX 怎么办`), 这里禁用的是正文段落里的标点。

### 1.3 节奏规则

- 长短句交替, 避免连续 3 句以上句式长度相近 (节奏呆板的信号)
- 全文至少 3 次出现"一句话独立成段"的断裂效果
- 段落偏离主线后必须有"扣主线句"拉回来

### 1.4 论述中的故意打破

让段落有"活人感"而不是教科书味:
- 重复强调 ("就是...就是...就是单纯的")
- 中途打断 ("我当时就...")
- 省略主语 ("想看看" 而非 "想要了解一下")
- 刻意模糊 ("反正就是那样")

### 1.5 活人感测试 (体感记忆 vs 知识性描述)

写情绪用体感记忆, 不用知识性描述:

| 知识性 (AI 味) | 体感 (活人感) |
|---|---|
| 我当时很震撼 | 我当时就愣住了 |
| 这非常感动 | 鼻子一酸 |
| 体验非常糟糕 | 我直接关了, 不想再开 |
| 效果非常好 | 跑出来那一刻我笑了 |

### 1.6 推荐口语化词组 (SEO 适配子集)

从卡兹克 skill 的口语化词组里, **剔除公众号味重 / 个人 IP 味重的**, 保留对 SEO 子虔小编姿态适用的:

**保留**:
- 转场: 坦率的讲 / 说真的 / 怎么说呢 / 其实吧 / 你想想看 / 回到 XX 这块 / 顺着上面的再聊聊
- 判断: 我是真的觉得 / 我自己的感受是 / 我觉得还是挺重要的
- 承认: 说实话 / 我自己也踩过坑 / 这个事儿我也踩过坑 / 说实话我也不确定 (注: 这条只 ME mode 用, DIRECTOR 不用)
- 情绪: 给我一下子整不会了 / 这一下给我懵了 (低频用, SEO 不能太情绪化)

**剔除 (公众号 IP 味)**:
- 大家也都知道 / 看到的兄弟可以把公屏打在弹幕上 → SEO 不与读者直呼
- 太特么赤鸡了 / 老阴逼 / 不是哥们 → 太口语 / 粗俗, SEO 工程师姿态不用
- 你敢信？？？ / 太离谱了 → 情绪强度过高
- 标点表情 `？？？` / `= =` / `。。。` → SEO 排版不用

### 1.7 L4 活人感终审 (SEO 改造版)

卡兹克 skill 的 L4 核心问题: "读完是一个有见识的普通人在认真跟我聊一件打动他的事, 还是一个 AI 在给我输出信息?"

SEO 改造版: **"读完, 是一个真用过 CAD 的工程师在认真讲, 还是一个 AI 在输出 SEO 模板?"**

这条用在工程化阶段的 humanizer L2 设计。MVP 阶段记录, 不实施。

## 2. 不迁移元素 (说明为什么)

| Kazik skill 元素 | 不迁移理由 |
|---|---|
| 文章原型 5 类 (调查实验 / 产品体验 / 现象解读 / 工具分享 / 方法论分享) | SEO 用 5 个 `intent_type` (learn/solve/choose/replace/understand), 是另一套分类体系 |
| HKR 选题质检 (Happy/Knowledge/Resonance) | SEO 是关键词驱动, 不做选题筛选, 关键词来了就写 |
| "绝大多数文章不加小标题" | SEO **必须**有 H2/H3 (搜索引擎可读性 + 用户 scannability + 命中长尾) |
| 公众号字数 4000-8000 | SEO 1500+ floor (见 `branch-original/DECISIONS_v3.md §A`) |
| 英雄之旅叙事弧 | SEO 文章是功能性的 (解决问题 / 比较选型), 不是叙事 |
| 文化升维 (连接到哲学 / 历史) | SEO 读者要答案, 不要文化升华 |
| 固定尾部 (作者署名 / 星标 / 三连) | SEO 用 BRIDGE_TO_PRODUCT + CTA (见 `inherited_rules.md` §H.1 Help-Don't-Sell) |
| "卡兹克本人" 第一人称姿态 | ZIXEL 用子虔小编 / 用过 CAD 的工程师 (见 `inherited_rules.md` §1 persona anchor + `CLAUDE.md`) |
| 读者直呼 ("屏幕前的你" / "你相信我") | SEO 文章不与读者过度互动, 子虔小编是工程师在讲事, 不是博主在面向粉丝 |
| 谦逊铺垫法 ("我也不知道行不行") | ME mode 可以借用 (软读 brand placement), DIRECTOR mode 不用 (DIRECTOR 是自信推产品) |
| 升番逻辑 (sketch 喜剧节奏) | SEO 比较类文章 (choose/replace) 可借用一点点, 但不是主线技巧 |
| 创意案例 / 微型故事 (亮出挑战→展示脑洞→秀过程→引爆结果) | SEO 不是评测文, 不需要案例叙事 |
| L3-5 文章原型专项检查 | 不适用 (SEO 没有这些原型) |
